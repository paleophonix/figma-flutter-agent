# Figma → Flutter Agent — обзор продукта

Краткий one-pager для продакта, дизайна и стейкхолдеров. Техническая документация: [README.md](../README.md), [docs/README.md](README.md).

---

## Что это

**Агент**, который берёт экран из Figma и генерирует **готовый Flutter UI** внутри **существующего** мобильного приложения: экраны, виджеты, тема, ассеты, опционально навигация.

Не конструктор «с нуля» — вы приносите Flutter-проект, агент наполняет `lib/` и `assets/`.

---

## Зачем бизнесу

| Боль | Как закрываем |
|------|----------------|
| Долгая вёрстка экранов по макетам | Автогенерация из Auto Layout + компонентов |
| 15–50 экранов — убиваем квоту Figma | Один dump файла → офлайн-генерация всех экранов |
| Дизайн правит макет каждый день | Инкрементальный sync: меняется только затронутое |
| Разработчик дописал логику в кнопке | Зоны `// <custom-code>` не затираются при регене |
| Дизайн шлёт правки в чат | GitLab Issue → превью в браузере → MR в `main` |
| «А на телефоне так же?» | Combat renders: Figma vs Flutter + heatmap |

---

## Что получает команда на выходе

- Экраны в `lib/features/<имя>/`
- Переиспользуемые виджеты в `lib/widgets/`
- Тема Material 3 или Cupertino (переключатель в конфиге)
- Иконки SVG, растровые картинки, шрифты в `assets/`
- Опционально: роутинг, табы, bottom nav, прототипные переходы из Figma

## Два режима работы

### 1. Локальный wizard (разработчик)

```text
figma-flutter  →  меню как у flutter run
```

Типичный путь:

1. **fetch** — скачать файл из Figma один раз  
2. **generate → batch** — сгенерировать все экраны без сети  
3. **select** — выбрать активный экран  
4. **launch** — быстрый просмотр на устройстве (без нового вызова LLM)

**По умолчанию** в меню — **launch**: самый быстрый способ «показать что есть».

### 2. Control panel (команда / процесс)

```text
Issue в GitLab + ссылка на Figma  →  job  →  ссылка на превью  →  MR
```

- Дизайнер или PM создаёт Issue с URL фрейма  
- Бот/воркер генерирует код и постит **ссылку на превью** в комментарий  
- После ревью — merge request в основную ветку  
- В комментариях: `/bug …` (починка), `/regen` (холодная регенерация)

Опционально: Discord `/generate`, REST API для своих интеграций.

## Ключевые фичи (коротко)

| Область | Возможности |
|---------|-------------|
| **Figma** | Один фрейм или весь файл; офлайн-кэш; дозагрузка только иконок или только JSON |
| **Генерация** | LLM понимает layout → детерминированный Flutter-код (не «сырой Dart от нейросети») |
| **Качество** | `flutter analyze`, repair при ошибках, 3700+ автотестов, corpus gates |
| **Визуал** | Сравнение с макетом, diff-heatmap, golden-тесты (опционально) |
| **Синк** | Повторная генерация не ломает кастомный код в отведённых зонах |
| **Мультипроект** | Несколько Flutter-приложений в одном workspace |

## Честные границы MVP

**Есть сейчас:**

- Layout, компоненты, тема, ассеты, адаптив  
- Прототипные переходы (fade/slide)  
- Batch по многим экранам  
- Превью для ревью  

**Не обещаем в базовом `generate`:**

- Идеальный pixel-perfect без итераций  
- Редактируемый текст 1:1 с Figma (шрифтовая идентичность = картинка, не Text)  
- Lottie и микроанимации слоёв  
- Бизнес-логика, API, стейт из Figma Variables  
- Автокоммит при каждом изменении в Figma (только через control panel / MR)

Подробнее: [reference/technical-notes.md](reference/technical-notes.md).

## Роли и сценарии

| Роль | Сценарий |
|------|----------|
| **Flutter-разработчик** | Wizard: fetch → batch → launch; итерации по одному экрану |
| **Дизайнер** | Issue + Figma URL; смотрит превью по ссылке; комментирует `/bug` |
| **Тимлид / release** | `signoff` перед релизом; CI с теми же гейтами |
| **Продакт** | Список экранов в `screens.yaml`; приоритет через Issues |

## Метрики успеха (предложение)

- Время от **нового фрейма** до **рабочего экрана на девайсе** (цель: минуты после первого dump)  
- **Доля экранов**, проходящих analyze без repair  
- **Визуальный diff** vs Figma на ключевых экранах (combat renders)  
- **Квота Figma**: dump-file на файл, не N вызовов на N экранов
