# Универсальная теория трансляции Figma → Flutter (формальная модель ядра)

> Роль: Lead Rendering Architect. Дата: 2026-06-05. Это НЕ список багов и НЕ UI-код — это
> математическая модель морфизма координатных алгебр и набор инвариантов для ядра (`ir_validate`).
> Опора: верифицированный код `parser/geometry.py`, `parser/text_line_height.py`,
> `parser/numeric_rounding.py`, `generator/layout_flex_policy.py`, `generator/ir_emitter.py`,
> `tools/ast_sidecar`. Эмпирические константы помечены `[k]`.

---

## 1. Фундаментальный диагноз — одна корневая слабость

**Пайплайн выполняет поэлементное лоссовое копирование свойств без сохраняемого координатного/
трансформационного инварианта.** Трансляция трактуется как *перенос данных*, а не как
*проверяемый морфизм* между двумя координатными алгебрами. Это распадается на четыре доказуемых греха
(все подтверждены кодом):

1. **Нет смены базиса.** Система живёт в абсолютных мировых координатах Figma, а эмитит в
   относительно-констрейнтное пространство Flutter через локальные эвристики. Округление
   **по каждому краю независимо** (`numeric_rounding.round_stack_placement`, 0.1px) ломает закон
   сохранения: `round(left)+round(width)+round(right) ≠ round(parent)`.
2. **Аффинная амнезия.** `geometry.transform_context_from_figma_node` декомпозирует полную матрицу
   (rotation, scale_x, scale_y, tx, ty), но потребляется **только** `rotation_degrees_from_figma_node`
   → в дерево попадает **один скаляр-угол**; scale/skew/translate и каскад предков выброшены. Поворот
   эмитится `Transform.rotate(alignment: topLeft)` поверх координат `absoluteBoundingBox` (повёрнутого
   AABB) — неверный pivot в неверном пространстве.
3. **Наивность единиц движка.** Параметры рендера целевого движка копируются «как есть», без инверсии
   его модели: Figma-blur → `BoxShadow.blurRadius` напрямую, хотя Flutter применяет
   `σ = 0.57735·blurRadius + 0.5`; текст без `StrutStyle`.
4. **Валидация без законов.** `ir_validate` проверяет структурную безопасность (no-crash: bounds,
   scroll, ghost), но **ни одного геометрического инварианта** — ни сохранения, ни базовой линии, ни
   композиции трансформаций.

**Итог:** отсутствует сохраняемый инвариант «эмитированное Flutter-дерево есть точная смена базиса
мирового графа Figma». Пиксельный дрейф — не сумма частных багов, а следствие этого отсутствия.

## 2. Формальная модель

**Пространство F (мир Figma):** ℝ², абсолютные px, origin top-left. Каждая нода `n` несёт мировой
AABB `B_w(n) = (X, Y, W, H)` и **родительски-относительную** аффинную матрицу `M(n) ∈ Aff(2)`
(Figma `relativeTransform` = 2×3 `[[a c tx],[b d ty]]`).

**Пространство L (раскладка Flutter):** алгебра ограничений — нода получает `BoxConstraints` сверху,
возвращает `Size`; позиция либо через `Offset` (`Stack`/`Positioned`), либо через флекс-распределение
(`Row`/`Column`).

**Трансляция** `τ: F → L` обязана быть структуро-сохраняющим отображением с инвариантом `I`:
> мировой AABB любой ноды **восстановим** из эмитированного L-дерева с ограниченной,
> ненакапливающейся ошибкой ε.

Четыре подзадачи примирения = четыре вектора исследования: топология (absolute→constraint),
базовая линия (текстовый бокс), аффинность (каскад матриц), изоляция отрисовки (Z + repaint).

## 3. Теоремы трансляции (T1–T5) — инварианты для ядра `ir_validate`

### T1 — Закон аффинного каскада (мировой трансформ = упорядоченное произведение)

Для ноды `n` с предками `a₀(root) … a_k=parent`:
```
W(n) = M(a₀) · M(a₁) · … · M(a_k) · M(n)        (композиция, не коммутативна)
```
Figma `M(n)` родительски-относительна; её перенос `(tx, ty)` — это **истинное положение локального
origin (0,0) ноды в системе родителя**, а поворот/масштаб происходят **вокруг этого origin**.
Поэтому:

- **Позиция** ноды = `(tx, ty)` из `relativeTransform`, а **НЕ** top-left из `absoluteBoundingBox`
  (последний — AABB уже повёрнутого слоя; использование его + `Transform.rotate(topLeft)` даёт двойное
  смещение).
- **Pivot Flutter** `alignment = topLeft`, а трансформ = `compose(translate(tx,ty), R(θ), S(sx,sy))`
  как явная `Matrix4`.
- **AABB — производная для проверки:** `B_w(n)` должен восстанавливаться разворачиванием неповёрнутого
  бокса `w×h` матрицей `W(n)`.

**Инвариант (ir_validate):** `‖ expand(M(n), w, h) − B_w(n) ‖∞ ≤ ε`. На границе извлечения поддерева
(extracted widget) дочерний трансформ должен домножаться на накопленный `W(parent)`, иначе масштаб/
поворот/прозрачность предков теряются.

**Текущее нарушение (verified):** хранится только `θ`; `(tx,ty,sx,sy)` отброшены; pivot поверх AABB.

---

### T2 — Закон сохранения линейной протяжённости (без утечки округления)

Для любой оси контейнера с внутренней протяжённостью `P` и детьми протяжённости `cᵢ`, зазорами `gᵢ`,
паддингом `pad`:
```
Σ cᵢ + Σ gᵢ + pad = P            (закон сохранения оси)
```
Независимое округление краёв его нарушает. **Правило кумулятивного префикс-округления** (телескоп):
```
eᵢ = round( Σ_{j<i} raw_j )       // округляем абсолютную границу, не длину
cᵢ = e_{i+1} − eᵢ                  // длина = разность округлённых границ
```
Тогда `Σ cᵢ = round(P)` **точно**, а ошибка каждой границы ограничена `≤ 0.5px` и **не накапливается**.
Для `Positioned`: на каждую ось выбирать **не более одной свободной координаты**; зависимая =
`P − pos − size` (вычисляется, не округляется отдельно).

**Инвариант (ir_validate):** `∀ контейнер: |P − (Σcᵢ + Σgᵢ + pad)| ≤ 0.5px` (иначе repair).

**Текущее нарушение (verified):** `round_stack_placement` округляет `left/top/right/width` независимо
(0.1px), сохранение не гарантируется → суб-пиксельные щели/нахлёсты.

### T3 — Гравитация базовых линий (компенсация текста)

Дано (уже парсится, `tree.py:154-159`):
`glyph_top_offset = renderBounds.y − bbox.y` (leading сверху Figma), `glyph_height = renderBounds.height`,
`lineHeightPx`. Цель — совместить **первую базовую линию** независимо от платформы.

Flutter `TextPainter` ставит первый baseline на `leading_top + ascentₚ`, где при множителе `h` и
метриках шрифта `(asc,desc)` нормированных: высота строки `= fontSize·h`, leading
`= fontSize·h − (asc+desc)·fontSize`, распределяется `leadingDistribution`.

**Двухкомпонентная компенсация:**
1. **Пин line box** через strut: `height_strut = lineHeightPx / fontSize`, `forceStrutHeight: true`
   — фиксирует высоту строки под Figma независимо от шрифта.
2. **Компенсирующий top-паддинг** для совмещения глифа:
```
Δ_top = glyph_top_offset − leading_above_flutter
      = (renderBounds.y − bbox.y) − (lineHeightPx − Hₘ)·w_above
```
где `Hₘ = (asc+desc)·fontSize` (метрическая высота глифов), `w_above` — доля leading сверху
(`0.5` для even, метрическая для proportional). Эмитить компенсацию только при `|Δ_top| > ε`.

**Инвариант (ir_validate):** прогнозируемый первый baseline `(Δ_top + ascentₚ)` = Figma baseline
`(glyph_top_offset + asc·scale)` в пределах ε. Это **единственный платформонезависимый способ**
совмещения текста *до* растеризации глифов (остаточный предел — глиф-шейпинг).

**Текущее нарушение (verified):** `StrutStyle` не используется (grep=0); эмит только `height` +
`leadingDistribution.proportional`; `glyph_*` в `Text` не идут (лишь в input-паддинг/иконки).

---

### T4 — Инверсия единиц движка (blur/эффекты)

Любой параметр размытия выражается в единицах **целевого** движка инверсией его модели, не копированием.

- **Тень:** Flutter рисует с `σ = convertRadiusToSigma(blurRadius) = 0.57735·blurRadius + 0.5`.
  Figma-radius `r` соответствует Gaussian `σ_figma = k·r` `[k]` (≈0.5 по модели CSS/SVG).
  Значит:
```
blurRadius* = (k·r − 0.5) / 0.57735
```
- **Фильтр (layer/background blur):** `ImageFilter.blur(σ)` где `σ = k·r` напрямую.

`k` — **единственная эмпирическая константа всей системы**; пинится одним спайком, вшивается в единый
модуль единиц (владелец — `ir_validate`/units). Спред Figma→Flutter `spreadRadius` инвертируется
аналогично отдельным замером.

**Инвариант:** все блюр-параметры проходят через `U(r)`; прямой проброс `blurRadius=r` запрещён.

**Текущее нарушение (verified):** `_shadow_expr:145` `blurRadius:{effect.blur}` напрямую; layer-blur
рисуется фейковым `BoxShadow` (`_render_native_blur_vector`) вместо `ImageFilter`.

### T5 — Теорема разбиения отрисовки (static/dynamic с сохранением Z)

Дан z-упорядоченный (после `overlap_sweep`) список absolute-сиблингов; класс каждого:
`S` (статика: нет интерактивных потомков и анимаций) или `D` (динамика). RepaintBoundary меняет
**слои, не порядок краски**, поэтому:

> **Run-Length разбиение:** оборачивать в `RepaintBoundary` каждый **максимальный z-непрерывный run
> класса S**. Run валиден ⟺ ни один `D`-элемент не имеет z между min и max z этого run.

Это **грубейшее** (минимум границ) корректное разбиение, дающее нулевую repaint-связанность статики с
динамикой при точном сохранении стэкинга.

```
partition = RLE( sort_by_z(siblings), key = class )
∀ S-run R:  ∄ d∈D : z_min(R) < z(d) < z_max(R)
```

**Инвариант:** разбиение уважает z-тотальный порядок; `overlap_sweep` уже даёт порядок + классы
окклюзии — теорема лишь RLE-сворачивает их.

**Текущее состояние (verified):** RepaintBoundary только на скроллабельных (`layout_common`,
`layout_navigation`); статичный wallpaper не изолирован (FID-26 в reviewer-backlog).

## 4. Архитектурный план: куда вшить теоремы (Parse → Plan → Emit → AST)

| Стадия | Роль в теории | Изменение |
|--------|---------------|-----------|
| **Parse** | Каноническая мировая IR | Перестать выбрасывать аффинность: класть в IR полный `M(n)` (`transform_context_*` уже считает sx/sy/tx/ty), мировой AABB, текст-метрики (`glyph_*`). IR абсолютна и immutable. |
| **Plan / normalize** | Предвычисление инвариантов | Один проход: каскад `W(n)` (T1), кумулятивное префикс-округление (T2), baseline `Δ_top` (T3), RLE-разбиение слоёв (T5). Чисто, без побочных мутаций. |
| **`ir_validate`** | **Проверяльщик теорем** | Из «no-crash guard» в **инвариант-гейт**: проверять T1 (reproject≈AABB), T2 (сохранение ≤0.5px), T3 (baseline ≤ε), T5 (z-валидность run'ов). Нарушение → repair, не тихий проброс. |
| **Emit** | Чистая проекция F→L | ≤1 свободная координата на ось (T2); `Transform(Matrix4 compose, alignment: topLeft)` + позиция из `(tx,ty)` (T1); `StrutStyle`+`Δ_top` (T3); blur через `U(r)` (T4); RepaintBoundary по разбиению (T5). |
| **AST postprocess** | Идемпотентный синтакс | **Никакой геометрии.** Только нормализация синтаксиса/импортов/const; запрещено пере-округлять или двигать координаты. |
| **Units module (T4)** | Единственный владелец `k`, `U(r)` | Один источник истины коэффициента блюра; потребляется emit. |

**Принцип разделения:** Parse даёт истину (мир + метрики), Plan считает инварианты один раз,
ir_validate их **доказывает**, Emit **проецирует**, AST **не трогает геометрию**. Это устраняет
дивергенцию путей (deterministic/IR получают один нормализованный мир) и анти-паттернинг (теоремы
универсальны, без `if type==…`).

## 5. Единый инвариант корректности

```
τ корректна  ⟺  ∀ нода n:
    reproject( emit(n) ) ≈ε B_w(n)            // геометрия/трансформ (T1,T2)
  ∧ baseline( emit(n) ) ≈ε baseline_figma(n)  // текст (T3)
  ∧ blur( emit(n) ) = U(r_figma(n))           // эффекты (T4)
  ∧ z_order/layers сохранены                   // отрисовка (T5)
```

**Эмитированное Flutter-дерево — это смена базиса мирового графа Figma с ограниченной,
ненакапливающейся ошибкой**, а не набор локально-округлённых копий. Pixel-perfect ⟺ `ε → 0` для
формы/геометрии/эффектов; текст асимптотически стремится к границе `strut + Δ_top`, и единственный
непробиваемый остаток — глиф-шейпинг (требует растеризации TEXT-нод).
