## 1. Goal

Build an isolated **Python CLI** (`figma-flutter-agent`) that, from an existing Flutter project root, runs one command:

```bash
figma-flutter generate \
  --figma-url "https://www.figma.com/design/FILE_KEY/...?node-id=1-2" \
  --project-dir ../demo_app
```

and produces:

- `lib/features/<feature>/<feature>_screen.dart`
- `lib/theme/app_colors.dart`, `app_spacing.dart`, `app_theme.dart`
- `lib/widgets/<name>.dart` (deduplicated reusables)
- `assets/icons/`, `assets/images/` (+ `pubspec.yaml` updates)

**Architecture:** Hybrid pipeline â€” rule-based Figma parse â†’ Claude Sonnet 4.6 structured JSON â†’ Jinja2 Dart assembly. Theme files are **deterministic** (not LLM-generated).

## 5. Data contracts

### 5.1 CleanDesignTree (parser â†’ LLM)

```json
{
  "id": "1:2",
  "name": "OnboardingScreen",
  "type": "COLUMN",
  "padding": { "top": 16, "bottom": 16, "left": 16, "right": 16 },
  "spacing": 12,
  "sizing": {
    "widthMode": "FILL",
    "heightMode": "HUG",
    "width": null,
    "height": null
  },
  "alignment": { "main": "start", "cross": "stretch" },
  "style": {
    "backgroundColor": "0xFFFFFFFF",
    "borderRadius": 8,
    "textColor": null,
    "fontSize": null,
    "fontWeight": null,
    "borderWidth": null,
    "borderColor": null
  },
  "text": null,
  "vectorAssetKey": null,
  "componentRef": null,
  "children": []
}
```

**`type` enum:** `COLUMN` | `ROW` | `WRAP` | `STACK` | `TEXT` | `IMAGE` | `VECTOR` | `INPUT` | `BUTTON` | `CARD` | `CONTAINER`

**`widthMode` / `heightMode`:** `FIXED` | `HUG` | `FILL`

**`alignment.main` / `cross`:** `start` | `end` | `center` | `spaceBetween` | `stretch` | `baseline`

### 5.2 DesignTokens (parser â†’ Jinja theme)

```json
{
  "colors": [{ "name": "primary", "value": "0xFF6750A4" }],
  "typography": [{ "styleName": "titleLarge", "fontSize": 22, "fontWeight": "w400" }],
  "spacing": [{ "name": "md", "value": 16 }]
}
```

### 5.3 FlutterGenerationResponse (LLM â†’ generator)

```json
{
  "screenCode": "class OnboardingScreen extends StatelessWidget { ... }",
  "extractedWidgets": [
    { "widgetName": "PrimaryButton", "code": "class PrimaryButton extends StatelessWidget { ... }" }
  ]
}
```

**Claude API:** `output_config.format.type = "json_schema"`, `strict: true`. No colors/typography in LLM schema â€” theme is rule-based.

## 6. Module specifications

### 6.1 `figma/url.py`

- Parse `file_key` from `/file/`, `/design/` URLs
- Parse `node-id` query param; convert `123-456` â†’ `123:456`
- Raise `FigmaUrlError` on invalid input

### 6.2 `figma/connector.py`

- Async httpx client, header `X-Figma-Token`
- `fetch_nodes(file_key, node_ids)` â†’ `GET /v1/files/:key/nodes?ids=...` (no `geometry=paths`)
- `fetch_file_styles(file_key)` â†’ styles map from full file or nodes response
- `fetch_variables(file_key)` â†’ `GET /v1/files/:key/variables/local`; catch 403 â†’ return `None`
- Retry on 429: exponential backoff, max 3 attempts
- Rate-limit asset batches to â‰¤20 node IDs per `/images` call

### 6.3 `parser/tokens.py`

- Primary: variables API if available
- Fallback: dedupe colors from `fills`, map text styles from `style` + global `styles`
- Output `DesignTokens` Pydantic model
- Name sanitization: `Primary/Brand` â†’ `primaryBrand` (camelCase)

### 6.4 `parser/tree.py` + `parser/layout.py`

- Strip: `absoluteBoundingBox`, `relativeTransform`, hidden nodes, editor metadata
- Map Auto Layout fields per research Â§1 table
- `layoutPositioning: ABSOLUTE` â†’ mark node for Stack/Positioned; emit CLI warning if >30% nodes are absolute
- `parser/dedup.py`: `INSTANCE` nodes â†’ collect master `COMPONENT` id; flag as reusable widget candidate

### 6.5 `assets/exporter.py`

- Collect exportable node IDs (IMAGE, VECTOR, nodes with `exportSettings`)
- SVG for icons; PNG at scales `[1, 2, 3]` for raster
- Write to `assets/icons/`, `assets/images/2.0x/`, `assets/images/3.0x/`
- Return asset manifest for pubspec + CleanDesignTree `vectorAssetKey` / image refs

### 6.6 `llm/client.py`

- Model: `claude-sonnet-4-6` (configurable via env, default fixed)
- System prompt: Material 3 style guide from research Â§3 (12 rules)
- User message: compact JSON `{ cleanTree, tokens, assetManifest, featureName }`
- Validate response against Pydantic model mirroring JSON schema
- 1 retry on validation failure with error appended to prompt

### 6.7 `generator/renderer.py` + `writer.py`

- Jinja renders theme files from `DesignTokens`
- Wrap LLM `screenCode` / widget `code` in `screen.dart.j2` / `widget.dart.j2` with imports + auto-generated markers
- `writer.py`: backup target files to temp dir before write; restore on any failure
- Full overwrite of target paths

### 6.8 `generator/pubspec.py`

- ruamel.yaml load/merge (no regex string splice)
- Ensure `dependencies:flutter_svg` (latest compatible)
- Append `assets:` paths idempotently

### 6.9 `cli.py`

```bash
figma-flutter generate \
  --figma-url URL \
  --project-dir PATH \
  [--feature-name NAME] \
  [--dry-run] \
  [--verbose]
```

- Validate `project-dir` contains `pubspec.yaml`
- Load `.ai-figma-flutter.yml` from agent repo or `--config` path
- Exit codes: 0 success, 1 user error, 2 API/LLM failure

## 13. Risk register

| ID | Risk | Mitigation | Owner |
|----|------|------------|-------|
| R1 | Variables API unavailable (non-Enterprise) | Fallback to styles/fills | parser/tokens |
| R2 | Messy Figma (no Auto Layout) | Warning + Stack fallback | parser/tree |
| R3 | LLM schema compilation failure | Minimal schema (screenCode + widgets only) | llm/schema |
| R4 | Large frame token overflow | Subtree chunking (post-MVP if needed) | llm/client |
| R5 | pubspec.yaml corruption | ruamel.yaml structural merge | pubspec.py |
| R6 | Asset export missing exportSettings | Skip + warn; document in README | assets/exporter |
